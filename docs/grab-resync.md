# Ре-синк меню из Grab (квартальный)

Источник истины по составу меню и ценам доставки — каталог хозяйки в GrabMerchant.
Скрейпить food.grab.com **нельзя** (ToS Cl. 3.1.11/3.1.12); единственный законный
источник — экспорт, который хозяйка делает сама.

## Как хозяйка делает экспорт

GrabMerchant (портал merchant.grab.com, не приложение) → Menu → **Bulk Update** →
Download. Получается zip с CSV (`*ItemID,*ItemName,*Price,*CategoryName,…`).
Пошаговая инструкция на тайском — в `HANDOVER-th.md`, §8.

## Как обновить сайт по свежему CSV

1. `scripts/data/grab-item-map.json` — карта `Grab ItemID → файл в src/content/dishes/`.
   Она переживает переименования файлов и смену слагов: сверяйтесь по ItemID, а не по имени.
2. Для каждой строки CSV:
   - ItemID есть в карте → сверить цену (`*Price` → `price_thb`) и доступность
     (`*AvailableStatus`: `UNAVAILABLE_TODAY` не трогаем — это кнопка «стоп-лист» на день;
     `UNAVAILABLE_PERMANENTLY` → `available: false`).
   - ItemID нет в карте → новое блюдо: создать JSON (имя вида `EN (ไทย)` парсится на
     name.en/name.th; если имя только тайское — перевести EN вручную для стабильного слага),
     `source: "grab_export"`, `reviewedAt: null`, добавить пару в карту.
   - Файл есть, а ItemID из карты пропал из CSV → блюдо удалено в Grab: спросить хозяйку,
     прежде чем удалять с сайта.
3. Русские имена НЕ трогать при ре-синке (они ручные); новые блюда — переводить вручную,
   без машинных калек.
4. При смене слага старый добавить в `previousSlugs` (генератор редиректов подхватит).
5. `pnpm run validate:content && pnpm run build && pnpm run test:e2e` → commit → deploy.
